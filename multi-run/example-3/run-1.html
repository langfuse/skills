Context-overflow turn failure

P1 Reliability ≥2 / 79 · 2.5%

One turn hard-errored: AI_APICallError: prompt is too long: 1,371,258 tokens > 1,000,000 — a listObservations result was fed back and blew past the 1M limit; the user got no answer. A second turn ("create 1 dataset based on production data") returned <truncated due to size exceeding limit> — same oversized-payload mode.

◆ Root cause

Tool results (esp. listObservations/getObservation with full io) are re-injected un-capped; a large query balloons context until the model call fails.

✓ Proposed fix

Cap/paginate tool outputs (row limit + per-field truncation); enforce a token budget before the model call; on overflow, summarize or ask to narrow rather than error.

#5

Malformed tool arguments

P2 Tool / Agent 3 / 79 · 5 calls

5 ERROR-level tool calls, all model-generated input-validation failures: listScores with operator:"&lt;" (an HTML-escaped < leaking into args), and getObservationFilterValues/listObservations sending disallowed props → "must NOT have additional properties." The agent retries and recovers, but each wastes a round-trip.

◆ Root cause

(a) HTML-entity encoding of </> in the tool-arg serialization path; (b) tool schemas/prompt don't enumerate allowed operators or forbid extra keys, so the model guesses.

✓ Proposed fix

Fix HTML-escaping in the tool-arg path; tighten tool descriptions to list allowed operator enums and set additionalProperties:false with clear error hints.
