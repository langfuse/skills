## Verified non-issues (checked, held up)

- **Truncation** — QA output tokens max 2,295, answers end cleanly; no `finish_reason: length`. Clean.
- **Cost/tokens** — total $9.04, mean $0.034, max $0.166/trace; no runaway. gpt-5 input-heavy (p90 27k) but bounded. (Context bloat contributes to latency #5, not a standalone cost incident.)
- **Retrieval relevance** — `searchLangfuseDocs` returns **290 distinct doc URLs** over 239 calls with well-formed queries and topically matching Langfuse docs; not the "same few docs for every query" failure. Content-inspected, not just `ok`.
- **Language matching** — 19 Chinese-input QA turns, **0** answered in the wrong language (all answered in Chinese). Clean.
- **`out_of_scope` and `user-intent` evaluators** — comments confirm both correctly read the user message and classify sensibly; valid signals.
- **Score coverage** — 100% of product traces carry ≥1 score (though quality-of-answer scores are invalid per #1).
- **Safety/PII** — no leaked secrets/PII observed in sampled I/O; `is_cursing` clean.

## Suggested order of action

1. **#1** fix the two mis-wired response-quality evaluators (mapping) — until then, production answer quality is unmeasured. **#2** then schedule Correctness on prod.
2. **#3 / #4** propagate tool errors to ERROR level + guarantee a non-empty fallback answer (kills silent failures and the empty outputs together).
3. **#5** attack latency (cap tool rounds, stream, cache retrieval).
4. **#6 / #7** trace_name-in-Fast-Preview docs gap; tighten off-topic scope guard.
5. **#8–#11** hygiene: separate synthetic traffic, instrument root I/O for image/voice, add user-feedback capture, sanity-check is_cursing.

## Open questions

- Is the docs CDN / Inkeep 5xx (finding #3) a transient incident this week or chronic? Needs a longer window to tell.

- Non-QA apps (image 42, sentiment 21, voice 9) analyzed only at trace + spot-check level (low volume, and root I/O uninstrumented, #9).
