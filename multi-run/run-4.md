## Suggested order of action

1. **#1 Fix the "Language & style" evaluator mapping** — cheapest fix, unblocks your primary quality metric (currently 91% false-negative noise).
2. **#2 Constrain `getLangfuseDocsPage` to real URLs** — root cause that also drives #3 (empty answers), #4 (latency tail), #5 (token waste). One fix, four wins.
3. **#3 Add a terminal "always answer" guard** — stop shipping blank responses.
4. **#6 Attach a Correctness evaluator to production** — you currently cannot see if answers are right in prod.
5. **#4/#5 Latency + token trimming**, then **#7/#8** (scope/friction), then **#9/#10** hygiene.
